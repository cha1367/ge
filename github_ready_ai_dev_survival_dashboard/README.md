# 생성형 AI 전후 개발자 직종별 고용 생존성 판별 대시보드

## 주제

**생성형 AI 패러다임(2022년) 전후 데이터의 스택 변동성을 기반으로 한 개발자 직종별 고용 생존성 판별**

## 선정 배경

### 1. 기술 환경의 변화와 불확실성

최근 생성형 AI의 도입으로 인해 개발 생태계의 기술 스택은 예전보다 훨씬 빠르게 변화하고 있습니다. 이러한 기술적 변화는 직종마다 체감되는 정도가 다르며, 이는 개발자의 연봉이나 고용 안정성과 같은 현실적인 조건에 서로 다른 영향을 미치고 있습니다.

### 2. 분석의 필요성

기술이 빠르게 변할수록 개발자의 고용 환경이 불안정해질 것이라는 가설은 널리 퍼져 있습니다. 하지만 이를 실제 데이터로 검증하여, 구체적으로 어떤 직종이 기술 변화에 취약하고 어떤 직종이 상대적으로 안정적인지를 정량적으로 분석한 사례는 드뭅니다.

### 3. 연구 목적

본 프로젝트는 Stack Overflow Developer Survey 데이터를 활용해 직종별 기술 변동성을 수치화하고, 이를 실제 경제적 보상 및 고용 안정성 데이터와 비교합니다. 이를 통해 개발 직종을 위험도에 따라 분류하는 것을 목표로 합니다.

## 분석 개요

이 대시보드는 2021년과 2025년 Stack Overflow Developer Survey 데이터를 비교해 생성형 AI 확산 전후의 개발자 직종별 기술 스택 변동성을 산출합니다. 이후 연봉 중앙값과 고용 비율을 결합해 고용 생존성을 계산하고, 직종을 4사분면으로 분류합니다.

## 주요 화면

- **Executive Summary**: 전체 분석 요약과 핵심 지표
- **표1 기술 변화**: 전체 개발자 기준 2021년·2025년 대표 기술 Top10 비교
- **표2 기술 변동성**: 직종별 기술 집합의 교집합·합집합 기반 변동성 계산
- **표3 고용 지표**: 2025년 연봉 중앙값과 고용 비율 시각화
- **표4 4사분면**: 기술 변동성 × 고용 생존성 기준 직종 분류
- **원자료 테이블**: 계산에 사용한 주요 입력 데이터 확인

## 계산 방식

### 기술 변동성

```text
Volatility = 1 - (Intersection_Count / Union_Count)
```

- `Intersection_Count`: 2021년과 2025년 기술 목록에 공통으로 등장한 기술 수
- `Union_Count`: 2021년과 2025년 기술 목록을 합친 전체 기술 수
- 값이 높을수록 해당 직종의 기술 스택 변화가 크다고 해석합니다.

### 고용 생존성

```text
Employment_Viability = (Salary_Score_2025 + Regular_Score_2025) / (1 + Volatility)
```

- 연봉 점수와 고용 안정성 점수를 합산한 뒤 기술 변동성으로 보정합니다.
- 기술 변동성이 높을수록 동일한 연봉·고용 점수에서도 생존성 점수는 낮아집니다.
